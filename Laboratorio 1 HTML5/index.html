¿Cuál es la diferencia entre Internet y la World Wide Web?

El Internet es una red de computadoras conectadas mientras que la World Wide Web es un sistema de información de documentos de hipertexto enlazados entre sí a los cuales se puede acceder gracias a la existencia del internet.

¿Cuáles son las partes de un URL?

Un URL está compuesto principalmente de 3 partes:

  • Protocolo- Indica cómo conectarse
  • Dominio: Es el servidor
  • Ruta: Especifica la ubicación del recurso
  • Parámetros: Información adicional para identificar o filtrar recursos del servidor
  • Documento: Los archivos al igual que este que componen lo que se muestra

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Para pedir información de un recurso en específico
  • HEAD: Similar a GET, no contiene body en su return se usa para verificar presencia de un recurso antes de utilizar GET
  • POST: Para enviar informacion a un servidor
  • PUT: Actualizar un determinado recurso en el servidor
  • PATCH: Para aplicar una modificación temporal a un recurso
  • DELETE: Para eliminar un recurso en específico

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se debe utilizar POST para enviar la información, sin embargo esta debe ser encriptada o tener algún tipo de seguridad dado que se trata de una contraseña.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utilizaría el método GET para obtener información de los recursos

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Este código 200 significa que la ejecución se realizó con éxito sin ningún error.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

No, esto significa que el usuario intentó acceder a un recurso inexistente.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Si, dado que en este caso se trata de un error del servidor, es decir hay un problema con el código del desarrollador.

Sobre el ciclo de vida y desarrollo de los sistemas de información:

¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de un sistema de información engloba todas las actividades del proyecto:

  1. Diseño conceptual: Se determina la necesidad y se evaluan potenciales soluciones. Guía de las actividades del proyecto para enfocarse en objetivos y entregables de los respectivos avances
    • Identificación de necesidades
    • Análisis de factibilidad
    • Requerimientos del sistema
    • Especificaciones del sistema
    • Revisión del diseño conceptual
  2. Diseño preliminar del sistema: Se diseñan las funcionalidades de los subsistemas que darán soporte al sistema, se definen interfaces entre subsistemas así como una evaluación general de requerimientos
    • Análisis de funcionalidades
    • Asignación de recursos para requerimientos
    • Opciones del sistema
    • Diseño de modelos preliminares
    • Especificaciones del desarrollo
    • Revisión del diseño preliminar
  3. Diseño detallado y desarrollo Especificación de diseño e interfaces
    • Desarrollo de prototipos
    • Especificación de procesos
  4. Producción y desarrollo: Inicia desarrollo según especificaciones anteriores
    • Producción
    • Pruebas
    • Distribución del sistema y operaciones
  5. Uso y soporte: Se usa el sistema para sus propósitos definidos
    • Uso del sistema en ambiente de usuario
    • Modificaciones para mejoras
  6. Efectividad: Se evalúa constantemente correcto funcionamiento del sistema pata que esté en su eficiencia más alta

¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de desarrollo de un sistema de información se enfoca en realizar los requerimientos del producto. Los sistemas son desarrollados de mejor manera mediante un ciclo de actividades:

  1. Planificación: Prepara diseño, define alcance, metodología de desarrollo, designar roles, responsabilidades y plazos
  2. Análisis: Se definen requisitos que debe incorporar el sistema
  3. Diseño: Se define cómo se cumple con requerimientos, se opta por opciónes más efectivas o eficientes dependiendo de costos y complejidad
  4. Desarrollo: Marca inicio de producción
  5. Integración y pruebas: Se valida funcionamiento y detectan errores
  6. Implementación: Se instala tanto harware como software, se genera documentación y capacitación de uso del sistema
  7. Mantenimiento: Se realizan ajustes necesarios para mejorar rendimiento y asegurar funcionaiento óptimo del sistema
