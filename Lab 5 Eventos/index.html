Sabias que... versión Programación Orientada a Eventos

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Al utilizar JavaScript se puede verificar que los datos sean válidos con la finalidad de no enviarlos incorrectamente al servidor, de esta manera se puede proporcionar retroalimentación visual de mejor entenimiento para el usuario antes de que realmente suceda un error. De igual forma se evita requerir información por parte del servidor evitando tráfico.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Para desactivar la validación automática del navegador se puede escribir novalidate en el inspector de la página. Por esta razón es importante de igual forma validar los datos del lado del servidor.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

No todos quieren saltarse las validaciones a propósito por lo que realizarlas desde Javascript mejora la experiencia del usuario al presentar alertas de manera instantánea sin tener que esperar a que se envíen los datos incorrectamente. Se presenta retroalimentación para errores accidentales antes de que realmente sucedan.
